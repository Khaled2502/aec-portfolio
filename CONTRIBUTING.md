# Contributing to AEC Portfolio

Thank you for your interest in contributing to the AEC Portfolio project!

## How to Contribute

### Reporting Bugs

- Use the GitHub Issues page to report bugs
- Provide a clear description of the bug
- Include steps to reproduce
- Mention your environment (OS, Node version, etc.)

### Submitting Features

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/amazing-feature`
3. Commit your changes: `git commit -m 'Add amazing feature'`
4. Push to the branch: `git push origin feature/amazing-feature`
5. Open a Pull Request

### Development Guidelines

- Follow the existing code style
- Use TypeScript for type safety
- Keep components small and reusable
- Write meaningful commit messages
- Test your changes before submitting

### Code Standards

- Use ESLint: `npm run lint`
- Components should be in `src/components/`
- Utilities should be in `src/hooks/`
- Constants should be in `src/constants/`
- Translations should be in `src/translations/`

### Setup Development Environment

1. Clone the repository
2. `npm install`
3. `npm run dev`
4. Create `.env.local` from `.env.example`
5. Add your EmailJS credentials
